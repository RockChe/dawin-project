# 資料庫同步備份機制（含前端設定 UI）

## Context

Dawin Dash 目前使用 Neon PostgreSQL（8 張表），沒有任何備份機制。本方案建立：
- 前端備份設定頁面（填入憑證、設定排程、監看狀態）
- 自動化定時備份（Vercel Cron）
- 備份到 Google Drive + **獨立的 R2 帳戶**（非應用程式用的 R2）
- 不覆蓋策略：每次備份產生獨立檔案，檔名帶時間戳

## 架構總覽

```
┌─────────────────────────────────────────────────┐

│  │ 設定表單   │ │ 憑證設定  │ │ 備份按鈕        │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │ 備份歷史記錄（時間、狀態、大小、目標）         │ │
│  └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
          │                          │
          ▼                          ▼
   Server Actions              API Route
   (config CRUD)           POST /api/backup
          │                     │
          ▼                     ▼
    configTable            src/lib/backup.js
    (加密憑證)              (核心匯出 + 上傳)
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼

              (googleapis)         (backups/ 前綴)
```

## 新增 DB Schema

### backup_history 表

```javascript
export const backupHistory = pgTable('backup_history', {
  id: uuid('id').defaultRandom().primaryKey(),
  target: varchar('target', { length: 20 }).notNull(),   // 'gdrive' | 'r2'
  fileName: varchar('file_name', { length: 255 }).notNull(),
  fileSize: integer('file_size'),                         // bytes
  status: varchar('status', { length: 20 }).notNull(),    // 'success' | 'failed'
  error: text('error'),
  durationMs: integer('duration_ms'),
  tableCounts: text('table_counts'),                      // JSON string
  createdAt: timestamp('created_at').defaultNow().notNull(),
});
```

## 備份檔案格式與命名

- **格式**：JSON
- **命名**：`dawin-backup-2026-03-22T110000.json`（永不覆蓋）
- **保留策略**：UI 設定，預設保留最近 30 份

## configTable 備份設定 Keys

| key | 說明 |
|-----|------|
| `backup_r2_account_id` | 備份用 R2 帳戶 ID |
| `backup_r2_access_key` | 備份用 R2 Access Key（AES 加密） |
| `backup_r2_secret_key` | 備份用 R2 Secret Key（AES 加密） |
| `backup_r2_bucket` | 備份用 R2 Bucket 名稱 |
| `backup_gdrive_email` | Service Account Email |
| `backup_gdrive_key` | Service Account Private Key（AES 加密） |
| `backup_gdrive_folder` | Google Drive 資料夾 ID |
| `backup_enabled` | 啟用/停用自動備份 |
| `backup_frequency` | 每 N 小時備份一次 |
| `backup_targets` | 啟用的備份目標 `["gdrive","r2"]` |
| `backup_keep_count` | 每個目標保留幾份備份 |

## 實作步驟

1. `src/lib/crypto.js` — AES-256-GCM 加解密
2. `src/server/db/schema.js` — 新增 backupHistory 表 + 遷移
3. `src/lib/backup.js` — 核心匯出 + 上傳邏輯
4. `src/server/actions/backup.js` — Server Actions
5. `src/app/api/backup/route.js` — Cron + 手動 API
6. `vercel.json` — Cron 排程（每 4 小時）
7. `src/middleware.js` — 放行 /api/backup
8. `src/app/(admin)/backup/page.jsx` — 前端設定頁
9. `src/components/dashboard/Sidebar.jsx` — 導航項目
10. `scripts/restore.js` — CLI 還原腳本
11. `package.json` — scripts + googleapis 依賴
