## 你需要準備的

### 一、Google Cloud Service Account（備份到 Google Drive 用）

#### 步驟 1：建立 GCP 專案（如已有可跳過）

```bash
# 登入 Google Cloud
gcloud auth login

# 建立專案（或使用現有專案）
gcloud projects create dawin-backup --name="Dawin Backup"

# 設定為目前專案
gcloud config set project dawin-backup
```

#### 步驟 2：啟用 Google Drive API

```bash
gcloud services enable drive.googleapis.com
```

#### 步驟 3：建立 Service Account

```bash
# 建立 SA
gcloud iam service-accounts create dawin-backup \
  --display-name="Dawin Backup Service Account"

# 建立並下載 JSON 金鑰
gcloud iam service-accounts keys create sa-key.json \
  --iam-account=dawin-backup@dawin-backup.iam.gserviceaccount.com
```

金鑰下載後，你需要從 `sa-key.json` 取出兩個欄位：
- `client_email` → 填入前端設定頁的「Service Account Email」
- `private_key` → 填入前端設定頁的「Private Key」

#### 步驟 4：在 Google Drive 建立備份資料夾

1. 前往 [Google Drive](https://drive.google.com)
2. 建立一個資料夾（例如「Dawin Backup」）
3. 右鍵 → 共用 → 新增以下 email（給予「編輯者」權限）：
   ```
   dawin-backup@dawin-backup.iam.gserviceaccount.com
   ```
4. 從網址列取得資料夾 ID：
   ```
   https://drive.google.com/drive/folders/【這段就是資料夾ID】
   ```
   → 填入前端設定頁的「資料夾 ID」

### 二、Cloudflare R2 備份帳戶（備份到 R2 用）

> 注意：這是「備份專用」的 R2 設定，與應用程式使用的 R2 是分開的（可以是同帳戶不同 Bucket，也可以是不同帳戶）。

#### 步驟 1：建立 R2 Bucket

1. 登入 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 左側選「R2 Object Storage」→「Create bucket」
3. 命名：例如 `dawin-backup`
4. 地區：選最近的（Asia Pacific）

#### 步驟 2：建立 API Token

1. 在 R2 頁面點「Manage R2 API Tokens」
2. 「Create API token」
3. 設定：
   - 名稱：`Dawin Backup`
   - 權限：`Object Read & Write`
   - 指定 Bucket：`dawin-backup`（限制範圍）
4. 建立後記下：
   - **Access Key ID** → 填入前端設定頁
   - **Secret Access Key** → 填入前端設定頁

#### 步驟 3：取得 Account ID

1. Cloudflare Dashboard 右側欄 → 「Account ID」
2. 或者在 R2 頁面的 URL 中：`https://dash.cloudflare.com/【Account ID】/r2`
   → 填入前端設定頁的「Account ID」

### 三、在前端設定頁填入憑證

1. 啟動 dev server：`npm run dev`
2. 用 super_admin 帳號登入
3. 左側選單點「資料備份」（⟳ 圖示）
4. 依序填入：

**R2 備份設定：**
| 欄位 | 填入值 |
|------|--------|
| Account ID | Cloudflare Account ID |
| Access Key ID | R2 API Token 的 Access Key |
| Secret Access Key | R2 API Token 的 Secret Key |
| Bucket Name | `dawin-backup`（或你取的名字）|

→ 點「測試連線」確認成功 → 點「儲存」

**Google Drive 備份設定：**
| 欄位 | 填入值 |
|------|--------|
| Service Account Email | `sa-key.json` 中的 `client_email` |
| Private Key | `sa-key.json` 中的 `private_key`（整段貼入）|
| 資料夾 ID | Google Drive 備份資料夾的 ID |

→ 點「測試連線」確認成功 → 點「儲存」

**排程設定：**
| 欄位 | 建議值 |
|------|--------|
| 啟用自動備份 | 開 |
| 備份頻率 | 每天 3 次 |
| 備份目標 | 勾選 R2 + Google Drive |
| 保留份數 | 30 |

→ 點「儲存排程」

5. 點「立即備份」測試一次，確認備份記錄顯示成功

### 四、CLI 備份指令（本地使用）

```bash
# 備份到本地 backups/ 資料夾
npm run backup

# 如需 CLI 直接備份到 R2（需在 .env 加入 R2_BACKUP_* 變數）
npm run backup:r2

# 如需 CLI 直接備份到 Google Drive（需在 .env 加入 GOOGLE_* 變數）
npm run backup:gdrive
```

CLI 用的環境變數（如果需要從命令列備份，加入 `.env`）：
```
R2_BACKUP_ACCOUNT_ID=...
R2_BACKUP_ACCESS_KEY=...
R2_BACKUP_SECRET_KEY=...
R2_BACKUP_BUCKET=dawin-backup

GOOGLE_SERVICE_ACCOUNT_EMAIL=...
GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n..."
GOOGLE_DRIVE_BACKUP_FOLDER_ID=...
```

### 五、還原指令

```bash
# 從本地備份還原（先試跑）
node scripts/restore.js backups/dawin-backup-xxx.json --dry-run

# 確認無誤後執行還原（會清空現有資料！）
node scripts/restore.js backups/dawin-backup-xxx.json --confirm
```

> 還原後所有使用者密碼會重設為隨機臨時密碼，Console 會列出。使用者首次登入需更改密碼。
